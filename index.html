Real-Time OpenGL Path Tracer
Scott Kauker



Description

This project demonstrates rendering real-time path traced images in a GLSL shader to create interesting global lighting scenes.

For each pixel, a ray is shot out from the camera and bounced randomly around the scene, gathering light as it bounces off of light emitters. This is repeated many times, with the averaged resulting colors eventually resolving into a physically accurate and aesthetically pleasing image.

While not a bidirectional path tracer (rays are only shot out from the virtual camera), it still resolves quickly enough for simple scenes with large area light sources that it is plenty interactable, while still being visually interesting.

While I did not have the time for it, I'd like to make this project more expandable and performant, starting with implementing true bidirectional path tracing. Second, I would make scenes based on uniform data instead of hard coded. Finally, more shape types, and a proper BSDF for non-diffuse surfaces.

Features

  • Monte-Carlo ray selection with cosine-weighted hemisphere distributions
  • Adjustable samples per-pixel and ray bounce length
  • Optionally fully render a scene by pressing F, fully resolving in seconds
  • Supports spheres, planes, and kind of cylinders, each with their own properties, color, and light emittance
  • Optional gaussian blur for noise elimination
  • Allow a free-look camera to move around the active scene.
  • Scene can be adjusted through uniform inputs
  • Reflections
  • Uses OpenGL spec 3.2, so it is widely compatible with many consumer graphics cards
